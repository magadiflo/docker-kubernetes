# Sección 08: Docker - Comandos para manejar imágenes y contenedores

---

A continuación se mostrarán los distintos comandos que pueden usarse para trabajar con imágenes y contenedores.

---

# Comandos para imágenes

----

## Lista imágenes

````bash
$ docker image ls
REPOSITORY                               TAG             IMAGE ID       CREATED         SIZE
user-service                             latest          2d6575b585bc   32 hours ago    248MB
jenkins/docker                           latest          b5ad3d34c441   7 weeks ago     1.13GB
sonarqube                                lts-community   f1bde0249ec4   2 months ago    603MB
````

## Elimina una imagen (sin especificar tag)

La siguiente instrucción eliminó la imagen `user-service`. Como no especificamos la etiqueta, por defecto elimina el
`(TAG) latest` que es el tag por defecto. Si la imagen tiene contenedores ejecutándose mostrará mensajes de error y no
podrá eliminarse.

````bash
$ docker image rm user-service
Untagged: user-service:latest
Deleted: sha256:2d6575b585bce008a98875d3038780bf30273b7162fc11276dffe2929476bb6d
````

## Elimina una imagen (especificando tag)

La siguiente instrucción eliminó la imagen `user-service` con TAG `v1`. Si la imagen tiene contenedores ejecutándose
mostrará mensajes de error y no podrá eliminarse.

````bash
$ docker image rm user-service:v1
Untagged: user-service:v1
Deleted: sha256:2d6575b585bce008a98875d3038780bf30273b7162fc11276dffe2929476bb6d
````

## Elimina imágenes no utilizadas

Elimina todas las imágenes no usadas o inactivas (dangling images) de Docker. Las imágenes inactivas son imágenes que
no tienen etiquetas y no están asociadas con contenedores en ejecución o detenidos.

````bash
$ docker image prune
````
